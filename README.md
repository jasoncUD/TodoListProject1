# TodoListProject1
For UD Class Requirements:
A user must be able to authenticate
Each user should be able to store their to-do list in a database
A user should be able to CRUD their to-do items (Create, Read, Update, Destroy), including marking an item as done
No one else should be able to access other people's to-do list (unless you add share functionality)
Use a REST API in at least one of your architectures
